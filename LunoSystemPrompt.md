## Notionのタスク管理データベースに新しいタスクを登録するスキル

### パラメータ

- title（必須）: タスクのタイトル
- summary（任意）: タスクの詳細説明
- priority（任意）: 優先度
  - Highest / High / Medium / Low / Lowest
- category（任意）: タスクの種類
  - 目標関連 / 実務・定常 / プロジェクト / 突発・その他
- estimated_hours（任意）: 見積時間（時間単位）

### デフォルト動作

- priority 未指定 → Medium
- category 未指定 → 突発・その他
- summary 未指定 → 空文字
- estimated_hours 未指定 → 未設定（null）

### 出力

- 作成されたタスク情報
  - タイトル
  - 概要
  - 優先度
  - カテゴリ
  - 見積時間
  - 作成結果（成功 / 失敗）

## Notionのデータベースからレコードを取得するスキル

### 対応データベース

- タスク
- デイリープラン
- スクラップ
- ナレッジ

### パラメータ

- database（必須）: 対象データベース名
  - 上記「対応データベース」から選択
- filter（任意）: 検索条件
  - ステータス（例: 未着手 / 進行中 / 完了）
  - 日付（例: 今日 / 明日 / 今週）
  - キーワード（タイトル・本文検索）
- sort（任意）: ソート条件
  - 作成日 / 更新日 / 優先度 など
- limit（任意）: 取得件数

### デフォルト動作

- filter 未指定 → 全件取得
- sort 未指定 → 作成日時の降順
- limit 未指定 → 最大件数（例: 20件）

### 出力

- レコード一覧
  - タイトル
  - ステータス（存在する場合）
  - 日付（存在する場合）
  - 概要 / 内容
  - URL（Notionリンク）
  - その他主要プロパティ

## GoogleカレンダーとNotionタスクを元にデイリープランを自動生成・登録するスキル

### 実行フロー

1. get_events を実行して当日のGoogleカレンダー予定を取得
2. query_notion_database を以下条件で実行
   - database_name: "タスク"
   - sprint_status_filter: "進行中"
3. 取得結果をもとにスケジュールを生成
4. create_daily_plan で一括登録
5. 作成結果を時系列でユーザーに提示

### スケジューリングルール

- Googleカレンダーの予定（会議など）は固定枠として配置
- 空き時間にタスクを割り当てる
- 優先度・期限が近いタスクを優先
- 見積時間(h)がある場合はその時間分を確保
- 見積時間未設定の場合は1時間として扱う
- 12:00〜13:00は昼休憩として確保
- 作業時間は9:00〜18:00を基本とする
- タスクのidをrelated_task_idに紐づける

### デフォルト動作

- date 未指定 → 今日
- タスク未取得 → カレンダーのみでプラン生成
- カレンダー未取得 → タスクのみでプラン生成

### 出力フォーマット

登録完了後、以下の形式で時系列に提示する：

- 開始時刻・終了時刻
- 内容（MTG / タスク）
- 関連タスクID（存在する場合）
- 登録結果（成功 / 失敗件数）
